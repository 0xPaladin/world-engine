# Gas Giant Planet Type — Implementation Plan

## Overview
Add a `gasgiant` planet type rendered using Three.js Shading Language (TSL) via the `tsl-textures` library, which provides a procedural `gasGiant()` texture. The gas giant uses a `MeshPhysicalNodeMaterial` with `colorNode` set to the TSL gas giant output, rather than the existing colormap-based `ShaderMaterial`.

---

## Step 1: Install dependency

```bash
npm install tsl-textures
```

`tsl-textures` v3.0.4 has no runtime deps (peer: three). Already using Three.js r174, which supports TSL (`three/tsl` export path) with `WebGLRenderer`.

**Risk:** The npm README recommends `three/webgpu` + `WebGPURenderer`, but Three.js r174 NodeMaterial compiles TSL to GLSL for WebGL. If `MeshPhysicalNodeMaterial` with `colorNode` fails under `WebGLRenderer`, fall back to baking the gas giant output into a `DataTexture` and using the existing `ShaderMaterial` with a modified fragment shader.

## Step 2: Add `gasgiant` to planet type system

### `src/planet.js`
- Add `generateGasGiantMap()` function:
  - Single flat plate (like airless): `map.plate_r = [0]; map.r_plate.fill(0); map.plate_vec = [zero]; map.plate_is_ocean = new Set()`
  - Zero elevation, zero moisture for all regions/triangles
  - No rivers (no downflow, no flow)
  - `quadGeometry.setMap(mesh, map)` as usual
- Add `case 'gasgiant': return generateGasGiantMap()` in `generateMap()` switch
- No changes to getters/setters needed

### `src/main.js`
- Add `'gasgiant'` to the planet type dropdown array: `['earthlike', 'airless', 'barren', 'hostile', 'gasgiant']`

## Step 3: Create gas giant material

### `src/renderer.js`
Add imports:
```js
import { MeshPhysicalNodeMaterial } from 'three';
import { gasGiant } from 'tsl-textures';
```

Add a new module-level variable:
```js
let gasGiantMesh = null;
let gasGiantMaterial = null;
```

Add `createGasGiantMaterial(seed)` function:
- Create `new MeshPhysicalNodeMaterial({ roughness: 0.8, metalness: 0.0 })`
- Set `material.colorNode = gasGiant({ scale: 2, turbulence: 0.3, blur: 0.6, colorA, colorB, colorC, seed })`
- Return the material

Modify `rebuildPlanet()`:
- If planet type is `gasgiant`: create the geometry (position + index), create `gasGiantMaterial` via the new function, create `gasGiantMesh`, add to scene
- Clean up previous `gasGiantMesh` in the remove section
- For non-gasgiant types, remove `gasGiantMesh` from scene if present

Modify `updateColormapTexture()`:
- If type is `gasgiant`, skip (return early — no colormap needed)

### `src/shaders.js`
- No changes needed (gas giant uses its own material, not the ShaderMaterial)

---

## Step 4: Handle climate, rivers, overlays for `gasgiant`

### `src/main.js`
- `window.applyPopulation()` already returns early for non-earthlike types (checked via `planet.getPlanetType()`). Gasgiant will naturally get `null` population and cleared overlays.
- `rebuildAll()` → `rebuildRivers()` already returns early for non-earthlike (compares `planetType !== 'earthlike'`). No change needed.
- `applyClimate()` will run but `quadGeometry.applyClimate()` will no-op on zero elevation/moisture. Acceptable, but we should skip it for performance.
  - Modify `applyClimate()` to return early if `planet.getPlanetType() === 'gasgiant'`
  - Same for `rebuildAll()` — skip `applyClimate()` for gasgiant

### `src/renderer.js`
- `rebuildCloudSphere()` returns early for non-hostile types. Gasgiant will not get a cloud sphere. (Optional: could add banded cloud layer later.)

## Step 5: Gas giant GUI controls

### `src/main.js`

Add a new GUI folder `Gas Giant` that appears when type is `gasgiant`:

```js
const GAS_GIANT_DEFAULTS = {
    scale: 2,
    turbulence: 0.3,
    blur: 0.6,
    colorA: '#fff8f0',
    colorB: '#f0e8b0',
    colorC: '#afa0d0',
    seed: planet.getSeed(),
};
```

Add controls that call `rebuildAll()` on change (or more precisely, regenerate the gas giant material + mesh).

**Implementation approach:** Store gas giant params in a module-level object in `planet.js` (or just in `main.js`'s `PARAMS`). On any param change, call a function that rebuilds just the gas giant material without regenerating the mesh.

**For simplicity in v1:** On any gas giant param change, just call `rebuildAll()` (which regenerates the mesh + material). Since mesh generation is fast (no plates/elevation for gasgiant), this is acceptable.

The Climate folder controls (temperature, rainfall, waterLevel) should be hidden/disabled when gasgiant is selected. Approach:
- Add `fClimate.domElement.style.display = 'none'` on type change to gasgiant
- Restore `fClimate.domElement.style.display = ''` on other types

## Step 7: Handle colormap getData for gasgiant

### `colormap.js`
- `getData('gasgiant')` will fall through to the `default` case (earthlike colormap). This is fine as a fallback, though the gasgiant material never uses the colormap texture. No change strictly needed, but add an explicit case returning `earthlikeColormap()` for clarity.
